Banco de tiempo

Introducción

Se va a detallar una aplicación para gestionar el tiempo solidario que cada usuario aporta a la comunidad asociada al banco de tiempo. Los usuarios invertirán una determinada cantidad tiempo en tareas que necesite la comunidad: de administración (mantenimiento de la comunidad) o para cubrir las necesidades de los usuarios. Dicha aplicación gestionará la cantidad de tiempo que cada usuario dedica a la comunidad. Los usuarios tendrán una cuenta de tiempo como si fuera una cuenta bancaria, y podrán disponer de dicho tiempo como forma de pago o cobro por unos servicios recibidos o prestados a la comunidad.

Un banco de tiempo es un sistema de intercambio de servicios o habilidades, en el que la unidad de intercambio no es el dinero habitual sino una medida de tiempo, por ejemplo, la hora. Se trata de ofrecer un servicio a cambio de unidades de tiempo, es decir únicamente se contabiliza el tiempo invertido en el servicio prestado para acumular esa ganancia en la cuenta de la persona que lo presta y que podrá canjear después para recibir otra contraprestación. Los tipos de servicios se pueden encontrar en un banco de tiempo podrían ser:

  • Atención y cuidado de personas
  • Cuidados personales
  • Tareas del hogar
  • Formación
  • Reparaciones
  • Orientación y asesoramiento

Propósito

Crear una aplicación que gestione la comunidad Banco de Tiempo, que funcionará como un banco pero la divisa será el VT (Virtual Time). Cuanto más tiempo dedique un usuario de la comunidad a ayudar al resto de usuarios, más VTs tendrá en su cuenta. Si por el contrario el usuario necesita tiempo de la comunidad, se le restará el tiempo solicitado de su cuenta.

Ámbito del Sistema

  • Nombre del sistema: Banco de tiempo.
  • Indicaciones del alcance:
    • El sistema debe dar información de todos los tipos de trabajo que estén disponibles.
    • Los datos de los usuarios deben ser tratados de forma correcta y asegurando suintegridad en todo momento.
    • Debe tener distinción de usuarios entre usuarios menores de edad y usuarios adultos.
    • El sistema debe ser cómodo para los usuarios.
  • Beneficios, objetivos y metas que se espera alcanzar:
    • Gran cantidad de usuarios en el sistema.
    • Satisfacción de los usuarios con el sistema.
  • Referencia a otros sistemas: Banco de tiempo de Tenerife Guaydil

Visión general del documento

Este documento describe el funcionamiento de la aplicación GITS para implementar una comunidad que se beneficia de las propiedades de un “banco de tiempo”. La aplicación cumplirá con una serie de requisitos funcionales impuestos por el cliente y otra serie de requisitos no funcionales que añadimos para un correcto funcionamiento de la aplicación. En el documento se contemplan los casos de uso y los diferentes roles que un usuario puede tener en la comunidad.

Descripción general

Funciones del Producto

Las funcionalidades de la aplicación será las siguientes:
  • Registro: datos de usuario, aceptación de condiciones, …
  • Creación de una “moneda virtual”: timecoin
  • Consulta de “saldo” cuenta y operaciones típicas de una cuenta bancaria normal: ingresos, gastos, transferencias (asociada a la nueva “moneda” virtual
  • Buscador de productos con filtros
  • Tablón de anuncios dividido en categorías
  • Panel de gestión de trámites: reclamaciones, cancelación de cuenta, …

Características de los usuarios

Consideramos 5 tipos de usuarios como podemos ver en la siguiente tabla:

Personajes y perfiles de usuario
Usuarios Descripción
Usuario adulto Será el usuario más común. Es el que sostiene la comunidad mediante mediante la realización de tareas o solicitud de servicios
Usuario tutelado Los usuarios menores de edad podrán realizar tareas en la comunidad siempre que alguno de sus tutores (cada usuario menor tendrá uno o dos tutores con perfil de usuario adulto, generalmente los padres) les autorice a realizar dichas tareas. Administrador El administrador llevará a cabo las tareas típicas de mantenimiento, actualizaciones, resolución de problemas técnicos, etc.
Gestor Usuarios con privilegios, estan autorizados para banear a otros usuarios que realicen actividades no autorizadas o ilícitas dentro de la aplicación (pornografía, fraude, etc), resolución de disputas entre usuarios, etc.

Requisitos funcionales

  • Registro: La APP dispondrá de un sistema de registro que solicitará: nombre de usuario, email, edad y número de teléfono (obligatorio para los usuario mayores de edad).
  • Verificación de Usuario: Se necesita saber los datos del usuario y así mostrar información relevante dependiendo del tipo de usuario que sea. Por ejemplo si fuera un usuario tutelado sólo se mostrarán trabajos que puedan realizar menores de edad o que sus tutores están de acuerdo que ellos realicen.
  • Cuenta de timecoins: al registrarse el usuario se le asignará una cuenta en el Banco de Tiempo donde podrá ver y gestionar el tiempo que ha ganado.
  • Catálogo de servicios: Existirá un catálogo de servicios que se pueden prestar a la comunidad, agrupados por secciones de afinidad.
  • Pasarela de pago interna: Se emulará una pasarela de pago para gestionar la contratación de servicios y “pagar” con cargo a la cuenta correspondiente del Banco de tiempo.
  • Gestor de trámites: Habrá una serie de formularios online “adhoc” para gestionar los diversos trámites inherentes a este tipo de plataformas: reclamaciones, solicitudes, cancelación de cuenta, tutela de menores no parentales, etc
  • Buscador: La aplicación debe contar con un buscador para que al momento de buscar algún tipo de trabajo se pueda filtrar entre los distintos tipos que hay.
